CSS Columns

CSS Multi-column Layout is a module of CSS that adds support for multi-column layouts. Support is included for establishing the number of columns, as well as how content should flow from column to column, gap sizes between columns, and column dividing lines (known as column rules) along with their appearance.

Properties

  • column-count: 3;
  • /* content is arranged into <integer> columns of equal size. column-count can also be set to auto, in which case the number of columns is determined by other CSS properties, such as column-width */
  • column-fill: auto;
  • /* controls how an element's contents are balanced when broken into columns (auto, balance, balance-all). only works when container has a set height */
  • column-gap: 5rem;
  • /* sets the size of the gap (gutter) between an element's columns */
  • column-rule-color: silver;
  • /* color of line between columns */
  • column-rule-style: dashed;
  • /* style of line between columns */
  • column-rule-width: 1px;
  • /* width of line between columns */
  • column-rule: dashed 1px silver;
  • /* shorthand for color, style, width */
  • column-span: all;
  • /* makes it possible for an element to span across all columns when its value is set to all. by default, doesn't work in firefox (can be enabled by the user but what user would ever know to do that?) */
  • column-width: 14rem;
  • /* specifies the ideal column width in a multi-column layout. the container will have as many columns as can fit without any of them having a width less than the column-width value */
  • columns: 14rem auto;
  • /* shorthand for setting column-width and column-count */

Breaks

There are various places we might want to control our breaks: breaks inside boxes, for example inside a figure element; breaks before and after boxes, for example a heading; and breaks between lines.

  • break-inside: avoid;
  • /* keeps the content inside a box from breaking (for example, applied to a figure with an img and fig caption)*/
  • break-before: column;
  • break-after: column;
  • /* forces a break before or after an element */
  • break-before: avoid;
  • break-after: avoid;
  • /* avoids breaking before or after an element */
  • orphans: 3;
  • widows: 3;
  • /* the orphans property controls the number of lines left on their own at the end of a fragment. the widows property controls the number left on their own at the start of a fragment. both take an integer as a value, which represents the number of lines to keep together at the end or start of a fragment. these properties only work inside a block container, such as a paragraph */
