CSS Shapes

float: left;
shape-outside: margin-box;

CSS Shapes are applied to floating elements. The specification defines a number of different ways to define a shape on a floated element but the end result is that it causes wrapping lines to wrap round the shape rather than following the rectangle of the element's box. The current specification states the element has to be floated in order to apply a basic-shape to it. This has the side-effect of creating a simple fallback for cases where the browser doesn't support CSS Shapes.

properties

  • shape-outside — defines the shape. the value can be one of four basic shapes, or, it can be set to the element's box model values: border-box, padding-box, content-box, margin-box, or, it can use a image that has transparancy.
  • shape-image-threshold — used with shape-outside: url() to set the threshold of image transparency used to define the shape area. If the value of shape-image-threshold is 0.0 (which is the initial value) then the area must be fully transparent. If the value is 1.0 then it is fully opaque.
  • shape-margin — sets a margin around the defined shape. Usually this works in tandem with a regular margin because the shape may get clipped by reference box's margin box.

shapes from images

You can use an image that has transparency to define the shape. Here, I've placed and floated and image, then added the property shape-outside: url('button.png'). In order for this to work though, the image has to be CORS compatible, which basically means that the shape-outside image must be on the same server as the web page. Unfortunately though, this doesn't translate when you're testing locally (CORS doesn't seem to like file:///). To work around this do one of the following:

  • use Safari and from the Develop menu, choose "Disable Local File Restrictions".
  • use python to run a simple local server from your testing directory with python -m SimpleHTTPServer OR python3 -m http.server, then access your files through http://localhost:8000/.

basic shapes

  • inset() - same as a default rectangle but allows you to inset (offset) the values, thus pulling the content in over the shape
  • circle() - creates a circle
  • ellipse() - creates an oval
  • polygon() - creates a polygon

Note: Firefox has an awesome feature in the developer tools that lets you edit a shape (or clip-path) in the browser so you can copy the new values to your css. Don't know if the other browsers have this too, but will look into it.

inset()

inset() takes values for top, right, bottom and left, plus a final value optional value for border-radius. For example, shape-outside: inset(0 50px 100px 50px round 0);.

circle()

circle() can accept two possible arguments. The first is the shape-radius and the second is the optional position. Radius can be a length or percentage but can also be one of the keywords closest-side or farthest-side. Position, if used, is set after they keyword 'at'. If omitted, the poistion will be set to center, however you can use any valid position here to indicate the position of the centre of the circle. For example, shape-outside: circle(50% at 50%);.

ellipse()

ellipse() works similar to the circle except that it takes two radius: x and y. It also can take an optional at position. For example, shape-outside: ellipse(30% 50% at left);.

polygon()

polygon() is the most complex. This shape accepts three or more pairs of values (a polygon must at least draw a triangle). These values are co-ordinates drawn with reference to the reference box. The Firefox dev tools are unbelievably helpful for working with this shape. For example, shape-outside: polygon(0 27px, 0 225px, 300px 300px);.

using pseudo-elements

You don't have to create an actual img or div to float and apply the shape-outside property to. You could create a pseudo-element instead with ::before. The main downside to this is that pseudo elements don't seem to work with the dev tools, so may not be as easy to fiddle around with but still a good option once your values are known.

combining with clip-path

The same basic shape values used for shape-outside can be used with clip-path. For example the image to the left has the properties: shape-outside: ellipse(40% 50%); clip-path: ellipse(40% 50%);
